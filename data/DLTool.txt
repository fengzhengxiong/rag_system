# 1 ALSON DLTool说明

ALSON DLTool是埃尔森智能科技自主研发的一款先进的深度学习软件，专门针对工业领域的图像处理需求设计。该软件集成了多种适用于工业应用的顶尖目标检测与实例分割深度学习算法，能够有效解决传统机器视觉技术难以应对的高难度问题，包括复杂的图像分割、精确的目标定位等任务。

ALSON DLTool拥有直观易用的操作界面，使得用户无需具备编程能力或深厚的深度学习理论知识，就能轻松完成模型的训练和验证工作。通过这款软件，用户可以便捷地进行模型开发，大大简化从概念设计到实际应用的整个过程。

## 1.1 核心算法模块
- **目标检测**: 用于识别图像中的对象及其位置。应用案例包括生产线上的自动缺陷识别，如检测产品表面的划痕或裂纹。
- **实例分割**: 不仅识别对象，还精确分割每个对象的轮廓。应用案例涉及精细分析，如在组件装配质量检查中区分并评估各个零件的装配准确性。

# 3 目标检测

## 3.1 算法介绍

​		目标检测（Object Detection）的任务是找出图像中所有感兴趣的目标（物体），确定它们的类别和位置，是计算机视觉领域的核心问题之一。

## 3.2 适用场景

​		定位工件、特征位置：

![](ALSON DLTool gif\obj_1.png)

![](ALSON DLTool gif\obj_2.jpg)

​		工件计数：

 ![](ALSON DLTool gif\obj_3.jpg)

 ![](ALSON DLTool gif\obj_4.jpg)

## 3.3 训练流程

![](ALSON DLTool gif\obj_flow.png)

### 3.3.1 新建项目
​		点击首页新建项目按钮，在弹出的窗口内填入项目名称、项目描述。

![](ALSON DLTool gif\obj_new_pro.gif)

### 3.3.2 目标检测

​		点击目标检测类型，之后点击新建按钮完成项目创建。点击项目列表中刚才新建的项目进入数据标注界面。

![](ALSON DLTool gif\obj.gif)

### 3.3.3 导入图像、数据集

​		导入图像是导入未标注的数据，导入数据集是导入已标注过的数据。

- **导入图像**: 打开图像所在文件夹，使用鼠标左键进行选择，按住Ctrl+鼠标左键进行多选，按Ctrl+A进行全选。

  ![](ALSON DLTool gif\import_pic_data.gif)

- **导入labelme数据集**: 数据集目录下需要有两个文件夹（JPEGImages和Annotations），JPEGImages中存放图像，Annotations中存放labelme标签文件（.json后缀），并且图像和标签需要一一对应。json中的shape_type只能为"rectangle"，如下：

  ```json
  {
      "label": "baigai",
      "points": [
          [
              598,
              363
          ],
          [
              1240,
              931
          ]
      ],
      "shape_type": "rectangle",
      "flags": {}
  }
  ```

  

  ![](ALSON DLTool gif\import_labelme.gif)

- **导入coco数据集**: 数据集目录下需要有一个JPEGImages文件夹和annotations.json标签文件。JPEGImages中存放图像。Json中含有的图像需在JPEGImages中存在。annotations.json中segmentation要是矩形框四个顶点的坐标，8个值，"segmentation": [[x1,y1,x2,y2,x3,y3,x4,y4]]，如下：

  ```json
  "annotations": [
          {
              "segmentation": [
                  [
                      600.3809523809524,
                      360.0,
                      600.3809523809524,
                      931.4285714285714,
                      1237.3333333333333,
                      931.4285714285714,
                      1237.3333333333333,
                      360.0
                  ]
              ],
              "iscrowd": 0,
              "image_id": 1,
              "bbox": [
                  600.3809523809524,
                  360.0,
                  636.9523809523808,
                  571.4285714285714
              ],
              "area": 363972.7891156462,
              "category_id": 1,
              "id": 1
          }]
  ```

  

  ![](ALSON DLTool gif\import_coco.gif)

- **导入VOC数据集**: 软件中目标检测项目中可以导入VOC数据集，数据集目录下需要有两个文件夹（JPEGImages和Annotations），JPEGImages中存放图像，Annotations中存放VOC标签文件（.xml后缀），并且图像和标签需要一一对应。

  ![](ALSON DLTool gif\import_voc.gif)

### 3.3.4 标注图像

​		目标检测使用矩形框进行标注，首先新建标签类别（只可使用字母和数字），鼠标右键点击右下角标签名可以修改标签名和标注框颜色。然后选中所要标注的标签类别，选择创建矩形，对目标进行标注后，鼠标左键点击保存按钮。

![](ALSON DLTool gif\obj_annote.gif)

​		快捷键：

![](ALSON DLTool gif\shortcut_key.png)

### 3.3.5 训练模型

​		所有图像标注完成后，点击数据集划分按钮对数据集按比例随机进行划分，默认训练集和验证集的比例为9：1，可根据需求自行调整，划分完成后可以右键点击图像列表中的图像进行训练集验证集切换。

![](ALSON DLTool gif\obj_split.gif)

​		确定划分后，点击训练按钮，可以点击开始训练使用默认参数训练，或根据需求自行调整。

![](ALSON DLTool gif\obj_train.gif)

​		在当前项目中，非首次点击开始训练时，会提示已存在数据集，是否重新划分。若在上次训练后对数据集图像进行了增删，或者对标注框、类别信息进行了修改，点击“确定”会使用最新的数据进行训练，点击“否”会使用上次训练使用的数据进行训练。

![train_data](ALSON DLTool gif\train_data.png)

### 3.3.6 验证模型
​		训练完成后，点击验证按钮，对验证集进行预测，并将结果可视化。

![](ALSON DLTool gif\obj_val.gif)

### 3.3.7 导出模型
​		点击导出按钮，导出后会有弹窗提示是否打开模型文件夹。

![](ALSON DLTool gif\obj_export.gif)

# 5 注意事项

## 5.1 图像质量要求
​		确保数据质量满足深度学习训练需求，避免因图像过曝、过暗、失真、模糊或遮挡等问题影响模型性能。同时，保证数据采集的背景、视角和高度与实际应用环境严格一致，以提高模型在现实世界中的应用效果。用户应在数据采集前确认实际应用条件，避免不一致性导致的重复工作和成本增加。

## 5.2 数据集要求

​		“实例分割”和“目标检测”模块通过分析图像中的物体特征来构建模型，并将其应用于实际环境。为了确保模型的高效性和准确性，必须采集与实际场景高度一致的数据集。

- **数据集采集**: 采集数据时需全面覆盖所有可能的物体摆放情况。例如，如果实际场景中物体可能横向、竖向或散乱堆叠，数据集也应包含这些情况的代表性图像。
- **选取合适的数据集**: 确保所选数据集反映了实际应用场景中物体的各种朝向、位置和相互关系，且一定要涵盖待检测目标的所有光照、颜色、尺寸等信息，尽量与终端现场保持一致。以保证模型能准确处理实际场景中的各种复杂情形。
- **图像数据数量、占比**: 训练集图像建议在50张以上，请勿大量使用重复的图像数据。训练集中不同种类图像的比例需平衡，不平衡的数据会导致模型偏向于频繁出现的类别，从而影响整体识别效率和准确性。例如，不应存在某一种物体有20张图像而另一种仅有3张的情况。

​		通过遵循以上原则，我们的模型才可以在实际应用中达到更高的识别效果和适用性，减少需要重复采集数据的风险。

## 5.3 标注要求

​		深度学习训练过程中，高质量的数据标注是必不可少的。为此，我们采用了以下标注策略以确保数据的精确性和应用的广泛性：

- **全面性**: 标注应包括所有符合条件的物体，避免任何遗漏。
- **正确性**: 每个物体的标注与其类别标签必须严格对应。
- **一致性**: 所有标注遵循同一规则，保持处理相似物体时的统一性。
- **精确性**: 标注应精确贴合物体边缘，防止标注不足或过度。
